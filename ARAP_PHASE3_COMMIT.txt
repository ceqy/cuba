## Summary
Implemented comprehensive domain layer with value objects, extended entities,
and business logic for all Phase 2 database features.

## Domain Layer Components

### Value Objects (value_objects.rs - 283 lines)
**Immutable value types:**
- `Money` - Amount with currency, supports zero/positive/negative checks
- `Currency` - Enum for CNY/USD/EUR/JPY/GBP + dynamic codes
- `DocumentReference` - Company code + document number + fiscal year
- `PaymentTerms` - Payment conditions with discount calculation logic
  - Calculate due dates
  - Calculate early payment discounts
- `CreditStatus` - Ok/Warning/Exceeded/NoLimit
- `ClearingType` - Full/Partial/Automatic/Net
- `PaymentMethod` - Wire/Check/ACH/Card

### Extended Entities (extended_entities.rs - 389 lines)

**Clearing Management:**
- `ClearingHistory` - Clearing document aggregate
  - Business logic: `is_balanced()`, `total_items_count()`
- `ClearingItem` - Individual clearing line items
- `ClearingType` enum

**Credit Management:**
- `CreditCheck` - Credit check execution records
- `CreditCheckResult` - Pass/Fail/Warning
  - Credit limit vs exposure comparison

**Payment Processing:**
- `PaymentProposal` - Payment run proposal aggregate
- `PaymentProposalItem` - Payment line items with discounts
- `PaymentProposalStatus` - Draft/Approved/Executed/Cancelled
- `PaymentRun` - Actual payment execution
- `PaymentRunStatus` - Created/Processing/Completed/Failed

**Dunning:**
- `DunningHistory` - Collection activity tracking
- `ContactMethod` - Email/Phone/Letter
- Dunning level support (1, 2, 3)

**Banking:**
- `BankAccount` - Partner bank account details
  - IBAN support
  - Primary account flag
  - Active/inactive status

**Advances:**
- `AdvancePayment` - Prepayment tracking
  - Business logic: `can_apply()`, `apply()`
  - Auto-status update on full application
- `AdvancePaymentStatus` - Active/FullyApplied/Cancelled

**Analytics:**
- `AgingBucket` - Age stratification value object
  - Methods: `total()`, `overdue_total()`
  - Buckets: Current, 1-30, 31-60, 61-90, 90+ days

### Module Organization (mod.rs)
- Exported all value objects
- Exported all extended entities
- Integrated with existing entities and repositories
